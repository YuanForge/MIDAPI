# 代理商站点功能简版说明

## 目标

新增代理商体系，让代理商可以快速搭建自己的独立站点，对外销售平台能力，同时主站继续掌握渠道、成本、价格和服务稳定性。

## 核心模式

- 代理商先生成平台 Key，才能创建代理站点。
- 代理站点创建完成后，会自动使用该 Key 对接主站 API。
- 代理站点可以实时同步主站模型、渠道和平台售价。
- 代理商不能查看或修改主站真实渠道进价。
- 代理商进价等于主站平台售价。
- 代理商只能设置利润倍率。
- 代理站终端售价 = 代理商进价 × 利润倍率。

示例：

```text
主站平台售价：0.80 元 / 秒
代理商利润倍率：1.70
代理站终端售价：1.36 元 / 秒
代理商利润：0.56 元 / 秒
```

## 隔离方式

每个代理站点独立运行，避免影响主站和其他代理商。

| 资源 | 隔离方式 |
| --- | --- |
| 代码 | 从主站代码复制一份独立目录 |
| 数据库 | 每个代理站独立数据库 |
| Redis | 每个代理站独立 Redis DB |
| NATS | 共用一台 NATS 服务，但按站点使用独立队列命名空间 |
| SMTP | 代理商填写自己的 SMTP |
| 站点品牌 | 代理商填写网站名和 Logo |

这样做的好处是：代理站点出问题不会影响主站，代理站点之间也不会互相串任务、串数据。

## 主站负责什么

- 管理代理商账号。
- 管理代理商 Key。
- 创建和管理代理站点。
- 维护真实渠道进价。
- 维护平台售价。
- 给代理站提供价格和渠道同步 API。
- 监控代理站使用情况。

## 代理站负责什么

- 展示代理商自己的品牌、网站名和 Logo。
- 使用自己的数据库和 Redis。
- 使用代理商 Key 调用主站 API。
- 同步主站平台售价作为代理商进价。
- 让代理商设置利润倍率。
- 面向终端客户销售服务。

## 一期建议范围

1. 管理后台新增代理商管理。
2. 代理商后台单独做一个 Web。
3. 代理商后台只保留两个核心功能：生成 Key、搭建代理站点。
4. 代理商必须先生成 Key，才允许搭建站点。
5. 搭建站点时自动复制代码、创建数据库、分配 Redis DB、生成配置。
6. 代理站自动绑定代理商 Key，对接主站 API。
7. 价格逻辑只开放利润倍率，不开放主站渠道进价。
8. NATS 使用同一台服务，但每个站点使用独立命名空间。

## 结论

这个方案可以保证代理商独立运营自己的站点，同时主站仍然掌握价格、渠道和成本控制权。代理商可以自主加利润，但不能改平台进价，也不能影响主站运行。
