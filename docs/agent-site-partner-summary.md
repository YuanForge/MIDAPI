# 代理商站点功能合伙人简版

## 一句话说明

我们做一个代理商体系，让代理商可以一键搭建自己的独立站点，对外销售我们的 API 能力。主站继续掌握渠道、成本、价格和稳定性，代理商只负责品牌展示和加利润销售。

## 核心逻辑

代理商必须先生成 Key，才能搭建代理站点。代理站建好后，会自动用这个 Key 对接我们的主站 API，同步模型、渠道和平台售价。

代理商账号目前只由我们管理员创建，不开放公开注册。一个代理商可以创建多个代理站点，同一个代理商 Key 也可以绑定多个代理站点。

价格规则很清楚：

```text
代理商进价 = 我们的主站平台售价
代理站终端售价 = 代理商进价 × 代理商设置的利润倍率
```

示例：

```text
我们的平台售价：0.80 元 / 秒
代理商利润倍率：1.70
代理站卖给终端用户：1.36 元 / 秒
代理商利润：0.56 元 / 秒
```

代理商不能看到或修改我们的真实渠道进价，只能设置自己的利润倍率。

## 产品形态

系统分三部分：

| 部分 | 使用人 | 用途 |
| --- | --- | --- |
| 主站管理后台 | 我们自己 | 管理渠道、价格、代理商和代理站 |
| 代理商控制台 | 代理商 | 生成 Key、一步步搭建代理站 |
| 代理站后台 | 代理商 | 管理自己的站点、利润倍率和展示渠道 |

代理商控制台建议单独做一个 Web，流程像向导一样：

1. 生成或选择 Key。
2. 填网站名称和 Logo。
3. 填 SMTP 邮件配置。
4. 设置默认利润倍率。
5. 确认配置。
6. 系统自动搭建代理站。
7. 展示搭建结果和访问地址。

代理商需要先把域名解析到服务器。宿主机 Nginx 和域名由我们人工配置，代理站容器内部的 Nginx 由系统自动配置。

## 隔离方案

每个代理站独立运行，避免影响主站和其他代理商。

| 资源 | 隔离方式 |
| --- | --- |
| 代码 | 从主站代码复制一份独立目录 |
| 数据库 | 每个代理站独立数据库 |
| Redis | 每个代理站独立 Redis DB |
| NATS | 共用一台 NATS 服务，但每个站点有独立队列命名空间 |
| SMTP | 代理商填写自己的邮件配置 |
| 品牌 | 代理商填写网站名和 Logo |

这样即使某个代理站出问题，也不会影响主站和其他代理站。

代理站服务会自动启动。后期代理站更新暂时由我们人工拉代码、build，并执行 Docker Compose 更新。Redis DB 超过默认 16 个后，由我们扩容 Redis DB 数量或新建 Redis 实例。

## 渠道策略

一期建议只做“托管代理模式”。

代理站只能使用我们主站同步过去的渠道和价格，不能像主站一样自己添加第三方渠道。

代理站渠道页应该是：

```text
同步渠道 / 利润设置
```

代理商可以做：

- 查看同步来的模型和渠道。
- 查看自己的进价。
- 设置利润倍率。
- 查看终端售价。

代理商不能做：

- 新增第三方渠道。
- 隐藏同步渠道。
- 修改上游 API 地址。
- 修改上游 Key。
- 修改我们的平台售价。
- 查看或修改我们的真实渠道进价。

代理商暂时不需要高级自营模式。未来如果要给少数大代理开放自建第三方渠道，可以作为“高级自营模式”单独设计，不放在一期。

## 我们掌握什么

- 真实渠道进价。
- 平台售价。
- 可售模型和渠道。
- 代理商 Key。
- 代理站搭建状态。
- 代理站价格同步权限。
- 域名和宿主机 Nginx 配置。

## 代理商掌握什么

- 自己的网站名称和 Logo。
- 自己的 SMTP 邮件配置。
- 自己的默认利润倍率。
- 自己终端用户的运营。

## 一期范围

1. 主站新增代理商管理。
2. 单独做代理商控制台 Web。
3. 代理商可以生成 Key。
4. 代理商必须有 Key 才能搭建站点。
5. 系统自动复制代码、创建数据库、分配 Redis DB、生成 NATS 命名空间。
6. 代理站自动绑定代理商 Key，对接主站 API。
7. 代理站实时查询 / 同步我们的渠道和平台售价。
8. 代理商只能设置利润倍率，不能修改进价。
9. 代理站渠道页单独改成“同步渠道 / 利润设置”。
10. 代理站创建失败后，自动清理已创建的数据库和目录。

## 结论

这个方案可以让代理商快速拥有自己的独立站点，同时我们仍然控制渠道、价格和成本。代理商有利润空间，但不能绕开主站，也不能影响主站稳定性。
